# Glossary

Harness: orchestration shell for agents, skills, tools, workflow order, policies, and review gates.

RCD: Requirement Control Data. A normalized and traceable requirement control record.

RKB: Requirement Knowledge Base. Approved or reviewable requirement items used as the source of truth for BD work.

Source Unit: deterministically parsed source block with file and location metadata.

Traceable Chunk: normalized source unit with content hash, lifecycle status, ambiguity flags, and source reference.

Ontology: controlled labels and mapping rules for requirement semantics.

BD Frame: target Basic Design section such as screen, API, DB, validation, business rule, or exception.
