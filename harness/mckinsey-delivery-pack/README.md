# McKinsey Delivery Pack — Requirement Phase + MVP Phase

## Purpose

This folder contains the management consulting delivery pack required to run the Super AI Transformation Platform program through the Requirement Phase and MVP Phase.

The pack is designed for an Engagement Manager / Project Manager to operate the program with CEO alignment, business-first diagnosis, controlled MVP execution, governance, stage gates, and KPI tracking.

## Program positioning

```text
We are not building an AI app.
We are building a governed AI transformation capability that turns business processes into repeatable, traceable, measurable, and human-controlled AI-assisted delivery workflows.
```

Initial beachhead use case:

```text
RD / QA / Meeting Notes
  -> RCD
  -> BD Mapping
  -> BD Draft
  -> Validation
  -> Human Review
  -> Excel Delivery Pack
```

## How to use this pack

```text
1. Start with Requirement Phase Playbook.
2. Use WBS to plan weekly execution.
3. Use Governance/RACI file to lock ownership and gates.
4. Use KPI/Risk Tracker from day one.
5. Move to MVP Phase only after Requirement Gate is passed.
```

## Stage-gate logic

```text
Gate R0 — CEO / sponsor alignment
Gate R1 — Business diagnostic complete
Gate R2 — Target workflow and MVP scope approved
Gate M1 — MVP design approved
Gate M2 — MVP build ready for pilot input
Gate M3 — MVP output reviewable and traceable
Gate M4 — Pilot readiness / scale decision
```

## Management principle

Every activity must connect to at least one of the following:

```text
- Business pain
- Target capability
- Operating process
- Governance requirement
- KPI
- Reusable platform asset
```

If an activity does not connect to these, it is likely technology-first noise.
